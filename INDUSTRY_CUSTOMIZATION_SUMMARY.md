## 房产 Agent 的 5 个工作场景

### 场景 1：客户初访（信息收集）

```
客户：我想在北京买房，但我是外地户口，不知道能不能买
       
Agent：
1. 理解需求 - \"您是想买一套自住房吗？预算大概是多少？\"
2. 政策查询 - [调用 policy_query(\"北京\")]
3. 知识引用 - \"根据北京最新政策，外地户口需要...\"
4. 建议方案 - \"建议我们这样推进：先确认您的购买资格...\"
```

### 场景 2：房产对比（专业评估）

```
客户：这套房要 500 万，比另一套多 50 万，为什么要买这套？

Agent：
1. 深度分析 - \"让我从 8 个维度给您对比一下\"
2. 工具调用 - [housing_evaluate、housing_estimate]
3. 数据展示 - \"这套的户型方正度更好...\"
4. 专业建议 - \"从投资角度我建议选择这套...\"
```

### 场景 3：异议处理（说服客户）

```
客户：怎样保证不被骗？

Agent：
1. 认同感受 - \"这是完全正当的顾虑\"
2. 专业方案 - \"我们的保护措施有...\"
3. 实际案例 - \"我之前处理过类似情况...\"
4. 具体步骤 - \"第一步我们到房管局查产权...\"
```

### 场景 4：交易规划（流程指导）

```
客户：从现在看房到住进去要多久？需要多少钱？

Agent：
1. 时间规划 - \"按我们这边的经验，通常是...\"
2. 费用计算 - [transaction_fee(500万，北京，120）]
3. 详细清单 - \"您需要准备首付...\"
4. 风险提示 - \"需要注意的是...\"
```

### 场景 5：投资建议（长期规划）

```
客户：现在买房会升值吗？该买吗？

Agent：
1. 市场分析 - \"根据过去 3 年数据...\"
2. 政策研判 - \"从政策角度看...\"
3. 投资计算 - \"假设年升 5%，5 年后...\"
4. 个性建议 - \"对您的情况，我建议...\"
```

## 效果评估指标

### 量化指标

| 指标 | 目标 | 衡量方法 |
|------|------|--------|
| 工具命中率 | > 80% | 有多少比例的问题被正确工具处理 |
| 知识库引用率 | > 60% | Agent 回答中有多少引用了知识库 |
| 首次满足率 | > 70% | 客户第一次就得到满意答案的比例 |
| 对话自然度 | > 4/5 | 用户评分（1-5 分） |
| 答复速度 | < 3秒 | 平均响应时间 |

### 定性指标

- 客户是否感到被理解？
- Agent 是否像行业专家？
- 有没有给出有深度的建议？
- 客户是否乐意继续对话？
- 客户是否会把 Agent 推荐给朋友？

## 常见陷阱与避免方案

| 陷阱 | 症状 | 原因 | 解决方案 |
|------|------|------|--------|
| Agent 不调用工具 | 回答错误或含糊 | 提示词未指导工具黐用 | 在提示词中明确说「当我提到价格时请用 housing_estimate」 |
| 回答不专业 | 缺乏数据支撑 | 知识库太少或 Temp 太高 | 增加知识库，降低 temperature 到 0.3-0.5 |
| 对话不连贯 | 忘记前面的内容 | 消息历史管理不当 | 确保 conversation.max-history ≥ 10 |
| 成本很高 | API 调用太频繁 | 工具调用过度、LLM tokens太多 | 优化 max-tokens（2048 足够）、减少工具调用 |
| 响应很慢 | 用户等待 > 5 秒 | 知识库查询慢、LLM 超时 | 启用缓存、降低 top-k 数字、优化网络 |
